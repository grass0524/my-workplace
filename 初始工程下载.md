## 用户输入

```text
$ARGUMENTS
```

在继续之前，你**必须**考虑用户输入（如果不为空）。

## 概述

本命令用于从Git仓库下载初始化工程模板，支持Common-admin后管平台工程和移动端工程两种模板。下载完成后**自动启动工程**，并询问是否将该工程设置为后续功能开发的基础工程。

**核心特性**：
- ✅ **展示模板列表**：展示可用的初始化工程模板
- ✅ **智能项目识别**：根据对话上下文或时间自动识别关联的Spec项目
- ✅ **Git下载**：从指定的Git仓库下载模板工程
- ✅ **认证处理**：如果需要Git认证，会中断让用户输入用户名和密码
- ✅ **移除Git仓库**：下载完成后**自动移除初始工程的Git仓库**（避免提交到初始工程仓库）
- ✅ **自动启动**：下载完成后**自动启动工程**（无需确认）
- ✅ **工程设置**：询问是否将该工程设置为后续功能开发的基础工程

**支持的模板**：
- `common-admin` - Common-admin后管平台工程（Vue3 + TypeScript + Ant Design Vue）
- `mobile` - 移动端工程（uni-app + Vue3 + Wot UI）

## 执行流程

### 阶段0：输入识别与模板展示

#### 0.1 识别用户输入

**输入类型识别**：

1. **指定模板名称**：用户提供了模板名称（如 `common-admin`、`mobile`）
   - 直接使用指定的模板名称
   - 跳过模板列表展示，直接进入下载阶段

2. **未指定模板**：用户未提供模板名称
   - 展示模板列表让用户选择

#### 0.2 识别项目名称

**项目名称确定规则**（按优先级）：

1. **如果用户指定了 `--name=<项目名>` 参数**：
   - 直接使用指定的项目名称

2. **第一优先级：根据当前对话涉及的Spec**：
   - 检查对话上下文中是否提到了Spec目录路径（如 `docs/specs/017-competitor-analysis-pvai/`）
   - 检查最近打开的文件是否在某个Spec目录下
   - 如果找到，从目录名称提取项目名称（去掉编号前缀）
   - 示例：`docs/specs/017-competitor-analysis-pvai/` → `competitor-analysis-pvai`

3. **第二优先级：根据时间判断Spec下最新的文件夹**：
   - 扫描 `docs/specs/` 目录下的所有子目录
   - 按修改时间排序，选择最新的文件夹
   - 从目录名称提取项目名称（去掉编号前缀）
   - 示例：`docs/specs/017-competitor-analysis-pvai/` → `competitor-analysis-pvai`

4. **如果无法识别项目名称**：
   - 生成与Spec无关的初始工程名称
   - 获取当前日期时间戳（格式：`YYYYMMDD`，如 `20250112`）
   - 项目名称：`initial-project-{时间戳}`
   - 示例：`initial-project-20250112`
   - **注意**：时间戳只包含年月日，不包含时分秒

#### 0.3 展示模板列表并处理选择

**模板列表**：

1. Common-admin后管平台工程（`common-admin`）
   - 技术栈：Vue3 + TypeScript + Ant Design Vue 4.x
   - Git：`http://gitlab.bangdao-tech.com/68597/liuhuanwei-COMMON_ADMIN.git`

2. 移动端工程（`mobile`）
   - 技术栈：uni-app + Vue3 + Wot UI
   - Git：`http://gitlab.bangdao-tech.com/68597/liuhuanwei-MOBILE.git`

**选择处理**：
- 支持序号（`1`/`2`）、模板名称（`common-admin`/`mobile`）或关键词匹配（如 `后管`、`移动端`）
- 验证选择的模板是否存在，不存在则提示重新选择

### 阶段1：检查目标路径

#### 1.1 确定下载路径

**路径确定规则**（按优先级）：

1. **如果用户指定了 `--path=<path>` 参数**：
   - 使用用户指定的路径

2. **如果用户指定了 `--name=<项目名>` 参数或已识别项目名称**：
   - 下载路径：`docs/frontend-projects/{项目名}-initial`
   - 示例：`docs/frontend-projects/competitor-analysis-pvai-initial`

3. **如果无法识别项目名称（使用时间戳生成）**：
   - 获取当前日期时间戳（格式：`YYYYMMDD`，如 `20250112`）
   - 下载路径：`docs/frontend-projects/initial-project-{时间戳}`
   - 示例：`docs/frontend-projects/initial-project-20250112`
   - **注意**：这是与Spec无关的独立初始工程

#### 1.2 检查目标路径是否存在

**路径检查**：

1. **如果目标路径已存在**：
   - 检查目录是否为空
   - 如果目录不为空：
     - 提示用户：目标路径已存在且不为空
     - 询问用户是否要：
       - **覆盖**：删除现有目录，重新下载
       - **重命名**：使用新名称（如 `{模板名称}-{时间戳}`）
       - **取消**：取消下载操作

2. **如果目标路径不存在**：
   - 创建目标目录的父目录（如果不存在）
   - 继续执行下载

### 阶段2：Git下载处理

#### 2.1 准备Git下载命令

**Git命令准备**：

根据选择的模板和下载路径准备Git克隆命令：

- **Common-admin后管平台工程**：
  ```bash
  git clone http://gitlab.bangdao-tech.com/68597/liuhuanwei-COMMON_ADMIN.git {下载路径}
  ```

- **移动端工程**：
  ```bash
  git clone http://gitlab.bangdao-tech.com/68597/liuhuanwei-MOBILE.git {下载路径}
  ```

#### 2.2 检查Git是否可用

**Git环境检查**：

1. **检查Git是否安装**：
   ```bash
   git --version
   ```
   - ✅ 如果成功返回版本号 → Git已安装，继续执行
   - ❌ 如果命令不存在 → Git未安装

2. **如果Git未安装**：
   - 提示用户安装Git：
     - **Mac**：`brew install git` 或从 https://git-scm.com/download/mac 下载
     - **Windows**：从 https://git-scm.com/download/win 下载安装
     - **Linux**：`sudo apt-get install git`（Ubuntu/Debian）或 `sudo yum install git`（CentOS/RHEL）
   - **等待用户安装完成后继续**（询问用户是否已安装）

#### 2.3 执行Git克隆（尝试无认证）

**首次尝试**：

1. **执行Git克隆命令**（不包含认证信息）：
   - 在后台执行Git克隆命令
   - 捕获输出，检测是否需要认证

2. **检测认证需求**：
   - 如果输出包含 `Username`、`Password`、`Authentication failed`、`fatal: could not read Username` 等关键词
   - → **需要Git认证**，进入阶段2.4

3. **如果克隆成功**：
   - → **跳过认证步骤**，进入阶段3

#### 2.4 处理Git认证（如果需要）

**⚠️ 重要**：如果需要Git认证，**必须中断执行**，让用户输入用户名和密码。

**认证处理流程**：

1. **检测到需要认证**：
   - 停止当前执行
   - 提示用户：

   ```
   ⚠️ Git仓库需要认证
   
   请提供Git用户名和密码：
   - 用户名：________________
   - 密码：________________
   
   或者，您可以在URL中包含认证信息：
   http://用户名:密码@gitlab.bangdao-tech.com/68597/liuhuanwei-COMMON_ADMIN.git
   ```

2. **等待用户输入**：
   - 等待用户提供用户名和密码
   - 用户可以通过以下方式提供：
     - 直接在对话中输入用户名和密码
     - 或者提供包含认证信息的完整Git URL

3. **用户提供认证信息后**：
   - 使用提供的认证信息重新执行Git克隆
   - **方式1**：在URL中包含认证信息
     ```bash
     git clone http://用户名:密码@gitlab.bangdao-tech.com/68597/liuhuanwei-COMMON_ADMIN.git {下载路径}
     ```
   - **方式2**：使用Git凭据助手（如果已配置）

4. **验证克隆结果**：
   - ✅ 如果克隆成功 → 进入阶段3
   - ❌ 如果克隆失败 → 显示错误信息，询问用户是否重试

---

### 阶段3：验证下载结果

#### 3.1 检查目录结构

**验证步骤**：

1. **检查目标路径是否存在**：
   - ✅ 检查下载路径是否存在
   - ✅ 检查是否为目录

2. **检查关键文件**：

3. **如果验证失败**：
   - 提示用户下载可能不完整

#### 3.2 显示下载成功信息

**成功提示**：

```
✅ 工程下载成功！

📦 模板：{模板名称}
```

### 阶段3.3：移除初始工程的Git仓库

**⚠️ 重要**：基于初始工程生成的项目不应该提交到初始工程的Git仓库，因此需要在下载完成后立即移除初始工程的Git仓库。

**执行步骤**：

1. **检查是否存在.git目录**：
   ```bash
   test -d {下载路径}/.git
   ```

2. **如果存在.git目录**：
   - 移除.git目录：
     ```bash
     rm -rf {下载路径}/.git
     ```
   - 验证移除结果：
     ```bash
     test -d {下载路径}/.git && echo "移除失败" || echo "移除成功"
     ```

3. **显示移除结果**：
   ```
   ✅ 已移除初始工程的Git仓库
   
   说明：
   - 基于初始工程生成的项目将使用独立的Git仓库
   - 后续如需发布预览，需要初始化新的Git仓库
   ```

**执行时机**：
- 在验证下载结果成功后立即执行
- 在自动启动工程之前执行
- 无需用户确认，自动执行

```

选项:
  --name=<项目名>        指定项目名称（用于生成下载路径：docs/frontend-projects/{项目名}-initial）
  --path=<path>          指定下载路径（覆盖默认路径）
  --no-set-base          下载后不询问是否设置为基础工程
  --overwrite             如果目标路径已存在，直接覆盖（不询问）

  common-admin  - Common-admin后管平台工程

  - 下载完成后会自动启动工程，无需确认

  /初始工程下载 mobile                             # 直接下载移动端工程并自动启动
  /初始工程下载 --path=./my-project               # 下载到指定路径并自动启动
  /初始工程下载 common-admin --overwrite        # 如果路径已存在，直接覆盖
```

## 注意事项

1. **环境要求**：
   - 需要安装Git工具（如果未安装，命令会提示安装方式）
   - 前端项目需要Node.js环境（启动时会自动检查）
   - 确保已安装Node.js和包管理器（npm/yarn/pnpm）

2. **Git认证**：
   - Git仓库可能需要用户名和密码认证
   - 如果需要认证，命令会中断让用户输入
   - 建议使用Git凭据助手保存认证信息，避免每次输入

3. **路径与依赖**：
   - 如果目标路径已存在，会询问用户是否覆盖（建议使用默认路径）
   - 大型项目的依赖安装可能需要几分钟（首次安装会显示进度）
   - 如果网络较慢，建议切换npm镜像源

4. **移除Git仓库**：
   - 下载完成后**会自动移除初始工程的Git仓库**（`.git`目录）
   - 基于初始工程生成的项目将使用独立的Git仓库
   - 后续如需发布预览，需要初始化新的Git仓库

5. **自动启动**：
   - 下载完成后**会自动启动工程**，无需用户确认
   - 服务器启动后需要手动停止（在终端按Ctrl+C）

5. **项目名称识别**：
   - 优先根据对话上下文识别关联的Spec
   - 如果无法识别，会根据时间选择最新的Spec目录
   - 如果完全无法识别，会使用 `initial-project-{时间戳}` 格式

6. **基础工程设置**：
   - 设置为基础工程后，会影响后续的 `/前端设计` 等命令
   - 可以在后续命令中手动指定其他工程路径覆盖设置
