# 功能规格：打工人工作桌面 (Worker's Desktop)

**版本**：1.0.0  
**状态**：草稿  
**最后更新**：2026-02-28

## 1. 功能摘要

### 核心功能概览

"打工人工作桌面"是一个集成了健康打卡、任务管理、节假日提醒、外语学习及情绪记录的轻量化 Web 网页应用。该工具旨在通过便捷的浏览器访问，帮助职场人士平衡工作与生活，提升工作效率的同时关注身心健康。核心模块包括："吾日三省吾身"（健康习惯打卡，采用简单计数制）、"待办清单"（任务管理与统计）、"放假倒计时"（基于内置节假日数据）、"今日单词"（基于内置词库的每日一词）以及"心情日记"（情绪记录与回顾）。

### 关键业务流程

1.  **每日健康打卡**：用户查看"吾日三省吾身"模块 → 点击增加喝水/如厕/运动次数 → 系统简单记录累计次数。
2.  **待办任务管理**：用户新增待办事项 → 勾选完成 → 系统自动将已完成项置灰并沉底 → 实时更新当日/本周的完成率统计。
3.  **单词学习与复习**：系统每日从本地词库随机展示一个单词 → 用户查看今日单词 → 点击小喇叭听发音（TTS） → 加入生词库 → 进入生词库模式进行默写复习。
4.  **心情记录**：用户打开心情日记 → 选择当日心情并输入备注 → 保存 → 在日历或时间轴视图中回顾心情趋势。

### 关键数据实体（快速参考）

| 实体名称 | 业务含义 | 关键属性（业务层面） |
| :--- | :--- | :--- |
| **健康打卡** | 用户的每日健康习惯记录 | 打卡类型（喝水/如厕/运动）、日期、状态 |
| **待办事项** | 用户的工作或生活任务 | 内容、状态（未完成/已完成）、创建时间、完成时间 |
| **节假日** | 国家法定节假日信息 | 节日名称、日期、放假安排 |
| **单词** | 每日推荐或收藏的英语单词 | 英文拼写、中文释义、发音、生词标记、掌握程度 |
| **心情日记** | 用户每日的情绪记录 | 心情类型、文本内容、日期 |

### 关键交互点（快速参考）

*   **待办列表交互**：勾选复选框即时改变任务状态并触发列表重排序（完成项沉底）。
*   **单词发音**：点击小喇叭图标播放单词发音。
*   **心情视图切换**：通过切换按钮在"时间轴"和"日历"两种视图间无缝切换心情记录展示。

## 2. 用户场景和测试

### US-1: 健康习惯打卡
**作为** 一名关注健康的职场人士，
**我想要** 简单记录每天的喝水、如厕和运动次数，
**以便于** 了解自己当日的基本健康行为。

*   **验收场景 1.1：增加计数**
    *   **前提**：今日"喝水"计数为 0。
    *   **操作**：用户点击"喝水"打卡按钮。
    *   **结果**：计数变为 1，系统无需强制设定目标。
*   **验收场景 1.2：查看统计**
    *   **操作**：用户查看"吾日三省吾身"模块。
    *   **结果**：清晰展示今日喝水、如厕、运动的累计次数。

### US-2: 待办事项管理
**作为** 一名忙碌的打工人，
**我想要** 清晰地管理每日待办事项并查看完成进度，
**以便于** 提高工作效率，避免遗漏重要任务。

*   **验收场景 2.1：完成任务**
    *   **前提**：列表中有一条未完成的待办事项。
    *   **操作**：用户勾选该事项前的复选框。
    *   **结果**：该事项文字变为灰色并添加删除线，自动移动到列表底部。
*   **验收场景 2.2：查看任务统计**
    *   **操作**：用户查看待办模块的统计栏。
    *   **结果**：准确显示"当日"和"本周"的待办总数、已完成数、未完成数。

### US-3: 节假日盼头
**作为** 一名期待假期的员工，
**我想要** 知道距离下一个法定节假日还有多少天，
**以便于** 调整工作心态，安排假期计划。

*   **验收场景 3.1：查看倒计时**
    *   **操作**：用户查看"放假倒计时"模块。
    *   **结果**：显示文案"距离 [下一个法定节假日名称] 假还有 [X] 天"。

### US-4: 英语碎片学习
**作为** 一名希望利用碎片时间提升自我的员工，
**我想要** 每天学习一个新单词并能复习生词，
**以便于** 积累英语词汇量。

*   **验收场景 4.1：学习今日单词**
    *   **操作**：用户点击单词旁边的"小喇叭"图标。
    *   **结果**：系统播放该单词的标准发音。
*   **验收场景 4.2：加入生词库**
    *   **操作**：用户点击"加入生词库"按钮。
    *   **结果**：该单词被标记为生词，可在生词库中查看。
*   **验收场景 4.3：默写复习**
    *   **操作**：用户在生词库中启动"默写模式"。
    *   **结果**：系统隐藏单词拼写，仅显示中文释义，供用户输入拼写进行验证。

### US-5: 情绪记录与回顾
**作为** 一名注重心理健康的用户，
**我想要** 记录每天的心情并能按时间回顾，
**以便于** 觉察自己的情绪变化趋势。

*   **验收场景 5.1：记录心情**
    *   **操作**：用户选择今日心情（如：开心、疲惫）并输入"今天项目上线顺利"。
    *   **结果**：心情日记保存成功。
*   **验收场景 5.2：切换视图回顾**
    *   **操作**：用户点击"日历视图"按钮。
    *   **结果**：心情日记列表切换为日历样式，日历格子上显示对应日期的心情图标。

## 3. 需求

### 3.1 功能需求

#### 模块一：吾日三省吾身
*   **FR-101**：系统应提供"喝水"、"如厕"、"运动"三个固定计数项。
*   **FR-102**：用户点击对应按钮即可增加一次计数，无上限限制。
*   **FR-103**：系统仅展示"今日"累计次数，无需复杂的历史趋势图表或目标进度条。

#### 模块二：待办 (Todo)
*   **FR-201**：用户可以输入文本创建新的待办事项。
*   **FR-202**：用户可以通过勾选复选框将待办事项标记为"已完成"或"未完成"。
*   **FR-203**：**完成项置灰沉底**：当任务被标记为完成时，系统应自动将其样式置灰（含删除线），并将其排序移动到所有未完成任务之后。
*   **FR-204**：**统计看板**：系统应实时计算并展示以下统计数据：
    *   当日待办数、当日已完成数、当日未完成数。
    *   本周待办数、本周已完成数、本周未完成数。

#### 模块三：放假倒计时
*   **FR-301**：系统应内置当前年度的国家法定节假日数据（JSON格式）。
*   **FR-302**：系统应自动计算当前日期距离下一个法定节假日的天数。
*   **FR-303**：系统应展示固定格式文案："距离 [节日名称] 假还有 [天数] 天"。

#### 模块四：今日单词
*   **FR-401**：系统内置英语高频词库（如 CET-4/6 或职场常用词），每日随机展示一个。
*   **FR-402**：**点读功能**：调用浏览器原生 TTS (Text-to-Speech) 接口朗读单词。
*   **FR-403**：**生词库管理**：用户可将今日单词一键加入"生词库"。
*   **FR-404**：**复习默写**：系统应提供生词复习功能，支持"默写模式"（隐藏英文，看中文写英文，或听音写词）。

#### 模块五：心情日记
*   **FR-501**：用户可录入当日心情，包括选择心情类型（如表情图标）和输入文本备注。
*   **FR-502**：**统计功能**：系统应统计不同心情类型的出现频率（如：开心 5天，焦虑 2天）。
*   **FR-503**：**多视图展示**：系统应提供两种展示视图：
    *   **时间轴视图**：按时间倒序列表展示心情日记。
    *   **日历视图**：在月历上以图标或颜色标记每日心情。

### 3.2 非功能需求
*   **NFR-001 (可用性)**：界面应简洁清新，符合"工作桌面"的定位，避免过度干扰工作。
*   **NFR-002 (持久化)**：所有用户数据（打卡、待办、生词、日记）应在本地持久化存储，确保关闭应用后不丢失。

## 4. 成功标准

*   **用户活跃度**：用户每日打开应用并完成至少一项（打卡/待办/背词/记心情）操作。
*   **任务完成率提升**：用户通过使用待办统计功能，能够感知并提升每日任务的完成率。
*   **单词记忆效果**：用户在生词库默写功能的正确率随时间推移逐步提升。

## 5. 关键实体

### 5.1 健康打卡记录 (HealthCheckIn)
*   **类型 (Type)**：枚举值（喝水, 如厕, 运动）。
*   **时间 (Timestamp)**：打卡的具体时间。
*   **日期 (Date)**：归属的日期（用于按天统计）。

### 5.2 待办事项 (TodoItem)
*   **内容 (Content)**：任务的具体描述文本。
*   **状态 (Status)**：枚举值（未完成, 已完成）。
*   **创建时间 (CreatedAt)**：任务创建时间。
*   **完成时间 (CompletedAt)**：任务被标记完成的时间（用于统计）。
*   **归属日期 (Date)**：任务所属的日期（用于"当日"统计）。

### 5.3 节假日 (Holiday)
*   **名称 (Name)**：节日名称（如：国庆节）。
*   **开始日期 (StartDate)**：放假开始日期。
*   **天数 (Duration)**：放假天数。

### 5.4 单词 (Vocabulary)
*   **拼写 (Spelling)**：英文单词拼写。
*   **释义 (Meaning)**：中文解释。
*   **音频源 (AudioUrl)**：发音文件链接或TTS标识。
*   **生词标记 (IsNewWord)**：布尔值，是否被加入生词库。
*   **掌握程度 (MasteryLevel)**：用于复习功能的熟练度标记。

### 5.5 心情日记 (MoodEntry)
*   **心情类型 (MoodType)**：枚举或图标ID（如：开心, 一般, 难过）。
*   **备注 (Note)**：用户输入的文本内容。
*   **日期 (Date)**：记录日期。

---

## 6. 假设与依赖

*   **假设**：用户浏览器支持现代 Web 标准（如 TTS API）。
*   **假设**：用户无需跨设备同步数据，数据仅存储在本地（LocalStorage/IndexedDB）。
*   **依赖**：无外部 API 依赖，所有功能离线可用。
