# US-2: 待办事项管理

**优先级**：High
**父文档**：[spec-index.md](../spec-index.md)

## 1. 用户场景和测试

**作为** 一名忙碌的打工人，
**我想要** 清晰地管理每日待办事项并查看完成进度，
**以便于** 提高工作效率，避免遗漏重要任务。

### 验收场景

*   **验收场景 2.1：完成任务**
    *   **前提**：列表中有一条未完成的待办事项。
    *   **操作**：用户勾选该事项前的复选框。
    *   **结果**：该事项文字变为灰色并添加删除线，自动移动到列表底部。
*   **验收场景 2.2：查看任务统计**
    *   **操作**：用户查看待办模块的统计栏。
    *   **结果**：准确显示"当日"和"本周"的待办总数、已完成数、未完成数。

---

## 2. 功能需求

### 模块二：待办 (Todo)

*   **FR-201**：用户可以输入文本创建新的待办事项。
*   **FR-202**：用户可以通过勾选复选框将待办事项标记为"已完成"或"未完成"。
*   **FR-203**：**完成项置灰沉底**：当任务被标记为完成时，系统应自动将其样式置灰（含删除线），并将其排序移动到所有未完成任务之后。
*   **FR-204**：**统计看板**：系统应实时计算并展示以下统计数据：
    *   当日待办数、当日已完成数、当日未完成数。
    *   本周待办数、本周已完成数、本周未完成数。

---

## 3. 相关实体

*   **待办事项 (TodoItem)**
    *   **内容 (Content)**：任务的具体描述文本。
    *   **状态 (Status)**：枚举值（未完成, 已完成）。
    *   **创建时间 (CreatedAt)**：任务创建时间。
    *   **完成时间 (CompletedAt)**：任务被标记完成的时间（用于统计）。
    *   **归属日期 (Date)**：任务所属的日期（用于"当日"统计）。
