# 错误检测与监控模块

---
**模块名称**: 错误检测与监控  
**版本**: 1.0  
**创建时间**: 2025-01-XX  
**用途**: 检测服务器启动和运行过程中的错误，自动修复常见错误，并持续监控服务器状态  
**调用方**: `设计预览.md` 命令  
---

## 📋 模块概述

本模块负责在服务器启动后检测各种错误（端口冲突、编译错误、依赖缺失、配置错误等），自动修复可修复的错误，并持续监控服务器运行状态，生成状态报告。

---

## 🔄 执行流程

### 阶段4：错误检测与修复循环

#### 4.1 启动服务器并检测错误

**启动后立即检测**：

1. **端口冲突检测**：
   - 检测错误信息：`EADDRINUSE`、`Port XXXX is already in use`、`address already in use`
   - 如果检测到，自动尝试下一个可用端口

2. **编译错误检测**：
   - 检测错误信息：`Error`、`Failed`、`SyntaxError`、`TypeError`、`ReferenceError`
   - 提取错误详情（文件名、行号、错误信息、错误类型）

3. **依赖缺失检测**：
   - 检测错误信息：`Cannot find module`、`Module not found`、`package not found`、`Cannot resolve`
   - 提取缺失的包名和文件路径

4. **配置错误检测**：
   - 检测错误信息：`Configuration error`、`Config file not found`、`Invalid configuration`
   - 检查配置文件是否存在和格式是否正确

5. **环境变量缺失检测**：
   - 检测错误信息：`Environment variable not set`、`process.env.XXX is undefined`
   - 检查`.env`文件是否存在

6. **浏览器端错误检测**（如果使用浏览器自动化工具）：
   - JavaScript运行时错误（通过控制台日志）
   - 网络请求失败（404、500等HTTP错误）
   - 页面渲染问题（关键元素缺失、样式错误）
   - 控制台警告和错误信息

#### 4.2 错误修复策略

**修复优先级**（按严重程度和可修复性）：

1. **端口冲突** → **立即自动修复** ✅
   - 检测当前端口
   - 自动切换到下一个可用端口（+1递增）
   - 重新启动服务器
   - 最多尝试10个端口

2. **依赖缺失** → **自动修复** ✅
   - 提取缺失的包名
   - 自动执行安装命令：
     ```bash
     npm install <package-name>
     # 或
     yarn add <package-name>
     # 或
     pnpm add <package-name>
     ```
   - 等待安装完成
   - 重新启动服务器
   - 如果安装失败，显示错误并询问用户

3. **环境变量缺失** → **尝试自动修复** ✅
   - 检查是否存在`.env.example`或`.env.template`
   - 如果存在，复制并创建`.env`文件
   - 如果不存在，创建基本的`.env`文件（包含常用变量）
   - 提示用户补充必要的环境变量值

4. **配置文件缺失** → **尝试自动修复** ✅
   - **Vue项目**：
     - 检查`vue.config.js`是否存在
     - 如果不存在，创建基本配置
   - **React项目**：
     - 检查`vite.config.js`或`webpack.config.js`是否存在
     - 如果不存在，创建基本配置
   - **TypeScript项目**：
     - 检查`tsconfig.json`是否存在
     - 如果不存在，创建基本配置

5. **常见语法错误** → **尝试自动修复** ⚠️
   - **缺少分号**：自动添加分号
   - **括号不匹配**：尝试修复括号
   - **引号不匹配**：尝试修复引号
   - **如果无法自动修复**：显示错误详情和位置，询问用户

6. **导入路径错误** → **尝试自动修复** ⚠️
   - 检查文件是否存在
   - 尝试修复相对路径（`./`、`../`）
   - 检查`node_modules`中的包路径
   - 如果无法修复，显示错误详情

7. **类型错误（TypeScript）** → **显示错误详情** ⚠️
   - 显示类型错误的位置和原因
   - 提供修复建议
   - 如果错误较多，建议用户先修复类型错误

8. **浏览器端错误** → **尝试自动修复** ⚠️
   - **JavaScript运行时错误**：
     - 检测控制台错误信息
     - 提取错误位置（文件名、行号）
     - 尝试修复常见的运行时错误（未定义变量、类型错误等）
     - 如果无法自动修复，显示错误详情和位置
   - **网络请求失败**：
     - 检测404、500等HTTP错误
     - 检查API路径是否正确
     - 检查后端服务是否启动
     - 修复API路径或提示用户检查后端服务
   - **页面渲染问题**：
     - 检测关键元素是否存在（通过DOM查询）
     - 检查样式是否正确加载
     - 修复资源路径或样式问题

#### 4.3 持续监控与自动修复

**监控机制**：
- 持续监控服务器输出日志
- 实时检测新的错误出现
- 如果检测到错误，立即进入修复流程
- 如果服务器崩溃，自动重启

**修复循环**：
```
启动服务器 → 检测错误 → 分析错误类型 → 尝试自动修复 → 重新启动 → 验证修复 → 继续监控
```

**成功标准**：
- ✅ 服务器成功启动（检测到"Local:"、"http://"或"ready"输出）
- ✅ 无编译错误
- ✅ 无运行时错误
- ✅ 可以访问页面URL（HTTP 200状态码）
- ✅ 页面正常渲染（关键元素存在，样式正确）
- ✅ 无浏览器控制台错误（如果能够检测）

**终止条件**：
- ✅ 页面成功加载，无错误 → **成功，保持运行**
- ❌ 无法自动修复的错误（需要用户介入）→ **暂停，等待用户修复**
- ⏹️ 用户手动终止（Ctrl+C）→ **停止服务器**

### 阶段6：状态报告与持续监控

#### 6.1 生成预览状态报告

**报告内容**：

```markdown

- 类型：[前端项目/HTML原型]
- 路径：[项目路径]
- 服务器类型：[开发服务器/静态HTTP服务器]
- 访问URL：[http://localhost:端口号]
- 浏览器：[Chrome/系统默认浏览器]

🔧 启动过程：
- 环境检查：[Node.js版本/npm版本/Chrome状态]

- 如果遇到新错误，系统会自动尝试修复

═══════════════════════════════════════════════════
```

#### 6.2 保持服务器运行与持续监控

**持续监控机制**：
- 保持服务器进程运行
- 持续监控服务器输出日志
- 实时检测新出现的错误
- 如果检测到错误，自动进入修复流程
- 如果服务器崩溃，自动重启

**监控的错误类型**：
1. **编译错误**：代码修改后出现的编译错误
2. **运行时错误**：页面运行时的JavaScript错误
3. **依赖错误**：新添加的依赖缺失
4. **配置错误**：配置文件修改后的错误
5. **网络错误**：API请求失败等
6. **浏览器端错误**（如果使用浏览器自动化工具）：
   - JavaScript运行时错误（控制台错误）
   - 网络请求失败（HTTP错误）
   - 页面渲染问题（DOM元素缺失、样式错误）

## 📝 模块使用说明

在主命令中，通过以下方式调用本模块：

```markdown
### 阶段4和阶段6：错误检测与监控

**执行模块**：`modules/错误检测与监控.md`

请读取并执行该模块中的所有步骤，包括：
- 启动后立即检测各种错误
- 按优先级自动修复可修复的错误
- 持续监控服务器运行状态
- 检测浏览器端错误（如果使用浏览器自动化工具）
- 生成预览状态报告

执行完成后，继续执行主命令的后续阶段。
```

## 🌐 浏览器错误检测（可选）

如果主命令需要使用浏览器自动化工具进行错误检测，可以：

1. **使用Puppeteer或Playwright**（推荐）：
   - 自动打开浏览器
   - 捕获控制台错误
   - 检测网络请求失败
   - 验证页面渲染

2. **使用浏览器扩展**：
   - 通过浏览器扩展API获取控制台日志
   - 检测页面错误

3. **手动检测**（备选）：
   - 提示用户手动打开浏览器
   - 提示用户检查控制台错误
   - 根据用户反馈进行修复

**浏览器错误检测流程**：
```
打开浏览器 → 加载页面 → 等待页面加载完成 → 捕获控制台错误 → 检测网络请求 → 验证页面渲染 → 报告错误
```
