## 阶段0：输入验证与初始化

### 0.1 验证输入

**输入格式**：
- 必须提供竞品demo的URL（`http://` 或 `https://`）
- 可选：指定要分析的模块范围（如"仅分析用户管理模块"）

**验证规则**：
- ✅ URL格式必须合法
- ✅ 必须能够访问（非404）
- ⚠️ 如果遇到登录页面，提示用户并提供解决方案

### 0.2 创建功能目录

**目录结构**：
```
docs/specs/{编号}-competitor-analysis-{竞品名称}/
├── spec.md                          # 功能规格文档
├── competitor-analysis.md           # 竞品分析报告
├── feature-inventory.md             # 功能清单
├── page-catalog.md                 # 页面目录
├── interaction-flows.md             # 交互流程记录
├── business-rules.md                # 业务规则提取
├── frame-analysis.json               # 逐帧分析记录（⭐ 新增）
├── coverage-report.md                # 覆盖度报告
├── screenshots/                     # 所有截图
│   ├── page-XXX.png
│   ├── tab-XXX.png
│   ├── dialog-XXX.png
│   └── frame-XXX.png
└── interaction-records/            # 交互记录
    ├── page-XXX-elements.json
    ├── page-XXX-interactions.json
    └── page-XXX-coverage.json
```

**命名规则**：
- 功能编号：从 `docs/specs/` 目录中找到最高编号，使用 N+1
- 功能名称：`competitor-analysis-{竞品名称}`（从URL或页面标题推断）

## 阶段1：竞品demo访问与页面发现

### 1.1 访问首页

**检查清单**：
- [ ] 使用 `browser.navigate(url)` 访问竞品demo URL
- [ ] 等待页面完全加载（5-10秒，确保所有动态内容加载）
- [ ] 使用 `browser.snapshot()` 获取页面结构
- [ ] 使用 `browser.screenshot({ fullPage: true })` 截取完整页面
- [ ] 保存截图到 `screenshots/page-001-home.png`
- [ ] 创建首页帧记录（frame-001）

### 1.2 区域扫描与元素提取

**执行流程**：
1. 对首页执行**区域扫描检查清单**（6个区域全部扫描）
2. 提取所有导航菜单链接
3. 提取所有页面内链接
4. 识别所有Tab标签（顶部Tab、内容Tab）
5. 建立初始元素清单

**输出**：
- 首页元素清单（JSON格式）
- 导航菜单结构
- 初始页面清单

### 1.3 静态页面发现

**检查清单**：
- [ ] 提取所有导航菜单链接（排除锚点链接）
- [ ] 提取面包屑导航中的链接
- [ ] 提取Tab标签页中的链接
- [ ] 提取所有 `<a>` 标签的 `href` 属性
- [ ] 提取按钮的 `onclick` 或 `data-href` 属性
- [ ] 提取路由跳转（SPA应用的路由链接）
- [ ] 去重和分类
- [ ] 建立页面层级关系

**输出**：
- 静态页面清单（包含URL、标题、类型、所属模块）
- 页面层级关系图

### 1.4 交互发现隐藏页面

**⚠️ 关键**：通过实际交互操作，发现所有需要点击按钮、打开对话框才能访问的页面。

**交互发现优先级队列**：
1. **优先级1**：右上角区域所有元素（必须优先执行）
2. **优先级2**：顶部Tab切换
3. **优先级3**：左侧菜单所有链接
4. **优先级4**：内容区域Tab切换
5. **优先级5**：按钮、表单、对话框
6. **优先级6**：表格操作、筛选器
7. **优先级7**：其他交互元素

**交互发现检查清单**：

**右上角区域交互发现**（⭐ 优先执行）
- [ ] 点击头像，发现下拉菜单或跳转页面
- [ ] 展开用户菜单，提取所有菜单项
- [ ] 测试每个菜单项，记录跳转目标
- [ ] 点击通知图标，发现通知列表或消息中心
- [ ] 点击设置入口，发现设置页面
- [ ] 点击帮助/反馈入口，发现帮助页面或反馈表单
- [ ] 测试其他右上角元素

**顶部Tab交互发现**
- [ ] 点击所有顶部Tab标签
- [ ] 每个顶部Tab内容作为新帧分析
- [ ] 测试顶部Tab关闭按钮（如有）

**按钮交互发现**
- [ ] 遍历当前页面的所有按钮
- [ ] 对每个按钮执行点击操作：
  - [ ] 如果弹出对话框 → 记录为"对话框帧"，提取对话框内容
  - [ ] 如果跳转到新页面 → 记录为新页面URL，加入待分析队列
  - [ ] 如果显示表单 → 记录为"表单帧"，提取表单字段
  - [ ] 如果展开下拉菜单 → 提取菜单中的所有链接
  - [ ] 如果显示侧边栏 → 提取侧边栏中的所有链接和按钮

**Tab标签交互发现**
- [ ] 点击所有内容Tab标签
- [ ] 每个Tab内容作为新帧分析
- [ ] 检查是否有嵌套Tab，递归执行

**表格行操作发现**
- [ ] 对表格的每一行，点击"编辑"、"详情"、"删除"等操作按钮
- [ ] 记录弹出的对话框或跳转的页面

**筛选器交互发现**
- [ ] 展开所有筛选器下拉框
- [ ] 测试所有筛选选项
- [ ] 观察筛选后是否显示新的操作按钮或链接

**递归发现机制**
- [ ] 对每个新发现的对话框/页面/帧，重复执行上述交互发现流程
- [ ] 建立发现队列，确保所有交互元素都被测试
- [ ] 使用已测试元素集合，避免重复测试

**输出**：
- 完整页面清单（包含静态链接和交互发现的页面）
- 交互发现记录（记录每个页面的发现方式）
- 待测试交互队列

## 阶段2：逐帧深度分析

**⚠️ 重要**：对每个发现的页面和帧，执行以下完整分析流程。

### 2.1 帧访问与内容提取

**检查清单**：
- [ ] 访问页面URL（如为新页面）
- [ ] 等待页面加载（5-10秒）
- [ ] 执行完整滚动检查：
  - [ ] 分段滚动（垂直和水平）
  - [ ] 多容器滚动
  - [ ] 双向滚动
  - [ ] iframe支持
- [ ] 获取页面快照和截图
- [ ] 创建帧记录（frame-XXX）

### 2.2 区域扫描与元素提取

**执行流程**：
1. 对当前帧执行**区域扫描检查清单**（6个区域全部扫描）
2. 提取静态元素（表格、表单、按钮、筛选器等）
3. 通过交互发现隐藏元素（展开折叠元素、打开下拉菜单、触发Tab切换等）
4. 建立元素清单

**元素清单格式**：
```json
{
  "frameId": "frame-XXX",
  "pageUrl": "页面URL",
  "elements": {
    "topArea": [],
    "topRightArea": [],
    "leftArea": [],
    "contentArea": [],
    "bottomArea": [],
    "floatingElements": []
  },
  "tabs": {
    "topTabs": [],
    "contentTabs": [],
    "nestedTabs": []
  },
  "buttons": [],
  "forms": [],
  "tables": [],
  "dialogs": [],
  "filters": []
}
```

### 2.3 Tab递归测试

**执行流程**：
1. 对当前帧的所有Tab执行**Tab递归测试机制**
2. 每个Tab切换后：
   - [ ] 创建新帧记录
   - [ ] 执行区域扫描检查清单
   - [ ] 提取Tab内容的所有元素
   - [ ] 测试Tab下的所有交互元素
   - [ ] 检查是否有嵌套Tab，递归执行

### 2.4 交互功能测试

**⚠️ 关键**：此阶段必须测试**所有**可交互元素，不允许跳过任何元素。

**测试执行原则**：
1. **必须测试原则**：所有在阶段2.2中发现的元素都必须被测试
2. **递归测试原则**：对话框、侧边栏等新出现的元素也要立即测试
3. **完整记录原则**：每个交互操作都必须记录结果
4. **状态恢复原则**：测试后尽可能恢复到原始状态

**交互测试检查清单**：

**右上角区域交互测试**（⭐ 优先执行）
- [ ] 头像测试（点击、下拉菜单、所有菜单项）
- [ ] 用户菜单测试（展开、所有菜单项、关闭方式）
- [ ] 个人中心入口测试（所有入口、跳转页面）
- [ ] 通知/消息测试（点击、通知列表、跳转页面）
- [ ] 设置入口测试（点击、跳转页面）
- [ ] 帮助/反馈测试（所有入口、跳转页面或对话框）
- [ ] 其他右上角元素测试（语言切换、主题切换、全屏等）

**按钮点击测试**
- [ ] 从元素清单中获取所有按钮
- [ ] 对每个按钮执行：
  - [ ] 获取点击前的页面状态
  - [ ] 点击按钮
  - [ ] 等待2-3秒，观察页面变化
  - [ ] 获取点击后的页面状态
  - [ ] 判断交互结果类型（页面跳转/对话框/表单/数据变化/提示消息/无变化）
  - [ ] 如果弹出对话框，测试完对话框后关闭
  - [ ] 标记按钮为"已测试"

**表单交互测试**
- [ ] 对每个表单执行：
  - [ ] 提取所有表单字段
  - [ ] 对每个字段执行验证测试（必填、格式、长度、类型）
  - [ ] 填写有效数据，测试提交行为
  - [ ] 测试取消/重置按钮
  - [ ] 测试所有表单按钮

**筛选器测试**
- [ ] 展开所有筛选下拉框
- [ ] 测试每个筛选选项（单个、多个、清除）
- [ ] 测试筛选组合
- [ ] 测试筛选器重置按钮
- [ ] 记录筛选后的页面变化

**表格操作测试**
- [ ] 排序测试（所有可排序列头）
- [ ] 分页测试（所有页码按钮、上一页/下一页、跳转指定页）
- [ ] 行操作测试（每行的操作按钮）
- [ ] 批量操作测试（选择多行、批量操作按钮）
- [ ] 表格内链接测试

**导航测试**
- [ ] 点击所有导航链接，记录跳转目标
- [ ] 测试面包屑导航，点击每个层级
- [ ] 测试返回按钮，记录返回路径
- [ ] 建立完整的页面跳转关系图

**对话框/模态框测试**（递归测试）
- [ ] 提取对话框中的所有元素
- [ ] 测试对话框中的表单
- [ ] 测试对话框中的所有按钮（递归）
- [ ] 测试对话框的关闭方式
- [ ] 如果对话框中有嵌套对话框，递归测试

**下拉菜单测试**
- [ ] 点击下拉按钮，展开菜单
- [ ] 提取所有菜单项
- [ ] 对每个菜单项执行点击，记录行为
- [ ] 测试菜单的关闭方式

**权限相关测试**（⭐ 新增）
- [ ] 识别按钮的禁用状态（disabled属性、样式判断）
- [ ] 记录禁用原因（权限不足、条件不满足、数据状态等）
- [ ] 测试不同权限下的功能可见性（如可能，切换账号测试）
- [ ] 记录权限相关的业务规则

**数据状态测试**（⭐ 新增）
- [ ] 测试空数据状态（清空筛选条件、删除所有数据等）
- [ ] 记录空状态的UI展示（空列表提示、空表格提示等）
- [ ] 测试加载状态（页面加载、数据加载、按钮loading等）
- [ ] 测试错误状态（网络错误、服务器错误、超时等）
- [ ] 记录错误提示的展示方式
- [ ] 测试大数据量场景（分页、虚拟滚动、性能表现）

**交互细节测试**（⭐ 新增）
- [ ] 测试悬停效果（hover状态、tooltip提示、hover菜单）
- [ ] 记录tooltip的内容和触发方式
- [ ] 测试键盘快捷键（如Tab、Enter、Esc、方向键等）
- [ ] 测试右键菜单（如有）
- [ ] 测试拖拽操作（如拖拽排序、拖拽上传等）
- [ ] 测试双击操作（如双击表格行展开详情）
- [ ] 测试长按操作（移动端场景）

**表单细节测试**（⭐ 新增）
- [ ] 测试表单联动（字段A变化影响字段B的选项/状态）
- [ ] 测试表单自动保存（如有）
- [ ] 测试表单重置后的状态（是否清空、是否恢复默认值）
- [ ] 测试表单验证的实时反馈（输入时、失焦时、提交时）
- [ ] 记录字段之间的依赖关系
- [ ] 测试表单的自动填充（浏览器自动填充、历史记录填充）

**表格细节测试**（⭐ 新增）
- [ ] 测试列宽调整（拖拽调整、双击自适应等）
- [ ] 测试列排序的多种状态（升序、降序、默认、多列排序）
- [ ] 测试列筛选（如有列筛选功能）
- [ ] 测试列显示/隐藏（列设置、列选择器）
- [ ] 测试表格导出功能（导出格式、导出范围）
- [ ] 测试表格内编辑（如有内联编辑功能）
- [ ] 测试表格行选择状态（单选、多选、全选、取消选择）
- [ ] 记录表格的固定列、冻结列功能

**搜索细节测试**（⭐ 新增）
- [ ] 测试搜索建议/自动完成（输入时显示建议）
- [ ] 测试搜索历史（如有历史记录功能）
- [ ] 测试搜索高亮（搜索结果中的关键词高亮）
- [ ] 测试模糊搜索（部分匹配、通配符等）
- [ ] 测试搜索清除（清除按钮、清空输入）
- [ ] 记录搜索的触发方式（输入时、点击搜索按钮、回车键）

**文件上传测试**（⭐ 新增）
- [ ] 测试文件类型限制（允许的类型、禁止的类型）
- [ ] 测试文件大小限制（单个文件、总大小）
- [ ] 测试上传进度显示（进度条、百分比）
- [ ] 测试上传失败处理（网络错误、文件过大、类型错误等）
- [ ] 测试批量上传（多文件选择、批量上传）
- [ ] 测试拖拽上传（如有）
- [ ] 记录上传成功后的处理（预览、下载链接等）

**时间相关测试**（⭐ 新增）
- [ ] 测试实时数据更新（如有WebSocket、轮询等）
- [ ] 测试定时刷新（自动刷新间隔、刷新倒计时）
- [ ] 测试时间选择器的各种场景（日期范围、时间选择、快捷选项）

- [ ] 测试时间格式的展示（相对时间、绝对时间）

- [ ] 测试不同屏幕尺寸下的布局（桌面端、平板、移动端）
- [ ] 记录响应式断点（布局变化的关键尺寸）
- [ ] 测试窗口大小变化时的响应（调整浏览器窗口大小）
- [ ] 记录移动端特有的交互（如侧滑、下拉刷新等）

**其他细节测试**（⭐ 新增）
- [ ] 测试页面刷新后的状态保持（筛选条件、Tab状态、滚动位置等）
- [ ] 测试浏览器前进/后退的处理（状态恢复、数据刷新）
- [ ] 测试多标签页之间的状态同步（如有）
- [ ] 记录错误提示的展示方式（Toast、Alert、Inline、对话框等）
- [ ] 记录成功提示的展示方式
- [ ] 测试确认对话框的各种场景（删除确认、操作确认等）
- [ ] 测试页面离开前的提示（如有未保存数据）

- [ ] 所有边界情况都已测试（权限、状态、异常操作等）

### 2.5 业务规则提取

**提取策略**：
1. **从侧边备注提取**（最高优先级）
2. **从对话框提取**
3. **从UI行为推断**
4. **从交互结果推断**

**记录格式**：Markdown文档，包含：
- 业务规则标题（按模块分组）
- 每个规则包含：规则名称、来源、内容描述、约束条件
