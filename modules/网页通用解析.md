## 📋 模块概述

本模块提供所有网页解析相关的通用流程，包括URL验证、页面访问、登录处理、错误处理、返回数据结构等。各具体解析模块应引用本模块的通用流程，只实现自己特有的业务逻辑。

---

## 🔄 通用流程

### 阶段 1: URL 验证

#### 1.1 基础URL格式验证

**验证规则**:
- ✅ 必须以 `http://` 或 `https://` 开头
- ✅ 必须包含域名或 IP 地址
- ❌ 拒绝纯文件路径（如 `file:///path/to/file`）

**实现**: 使用正则表达式 `/^https?:\/\/.+/` 验证

#### 1.2 特定URL格式检测（可选）

各模块可以在此基础上添加特定的URL格式检测：

- **禅道需求解析**: URL中必须包含 `zentao` 和 `story`（不区分大小写）
- **设计文件深度分析**: URL中可能包含 `mastergo.com`、`figma.com`、`figma.io` 等
- **在线网页解析**: 通用HTTP/HTTPS链接

**验证失败处理**:
- 返回错误：`{ type: 'INVALID_URL', message: 'URL格式不正确', url: url }`
- 提示用户重新输入

---

### 阶段 2: 页面访问

#### 2.1 使用 Browser 工具访问页面

**标准步骤**:
1. 使用 `browser.navigate(url)` 导航到目标 URL
2. 等待页面完全加载（默认3-5秒，复杂页面可延长）
3. 使用 `browser.wait(3000)` 确保内容加载完成
4. 使用 `browser.snapshot()` 获取页面快照

**等待策略**:
- 普通页面：3秒
- 复杂页面（包含大量动态内容）：5-8秒
- 设计工具页面（MasterGo/Figma）：5-10秒（需要等待图层加载）

#### 2.2 检查页面可访问性

**检查项**:
- ✅ 页面正常加载（HTTP 200）
- ❌ 页面404或无法访问 → 返回错误：`{ type: 'PAGE_NOT_FOUND', message: '页面不存在', url: url }`
- ❌ 网络错误 → 返回错误：`{ type: 'NETWORK_ERROR', message: '网络连接失败', url: url }`
- ⚠️ 页面需要登录认证 → 进入登录处理流程（见阶段3）

---

### 阶段 3: 登录处理

#### 3.1 登录检测

**检测方法**:
- 检查页面是否包含登录相关元素（用户名输入框、密码输入框、登录按钮）
- 检查页面URL是否包含登录相关路径（如 `/login`, `/signin` 等）
- 检查页面文本内容是否包含"登录"、"Login"、"Sign in"等关键词
- 检查页面是否显示权限错误（如401、403错误提示）

#### 3.2 登录处理策略

根据不同的解析模块，采用不同的登录处理策略：

**策略A：等待用户手动登录**（适用于禅道需求解析）
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
检测到页面需要登录
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

当前URL: [URL地址]
该页面需要登录才能访问。

请您在浏览器中手动完成登录操作，登录完成后请告诉我"已登录"或"登录完成"。

提示：我会等待您完成登录后再继续提取需求内容。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**策略B：提示用户提供替代方案**（适用于在线网页解析）
- 提示用户提供替代方案（截图/本地HTML/公开链接）
- 返回错误：`{ type: 'LOGIN_REQUIRED', message: '页面需要登录，请提供替代方案', url: url }`

**策略C：通常不需要登录**（适用于设计文件深度分析）
- 设计工具通常使用公开分享链接，不需要登录
- 如果遇到登录页面，提示用户检查分享链接是否正确

#### 3.3 验证登录结果

**验证步骤**:
1. 等待用户确认登录完成（策略A）
2. 使用 `browser.snapshot()` 重新获取页面快照
3. 检查页面是否成功加载目标内容
4. 检查是否仍显示登录页面（如果仍显示，提示用户重新登录）

---

### 阶段 4: 页面内容获取

#### 4.1 获取页面快照和截图

**标准操作**:
- 使用 `browser.snapshot()` 获取页面结构快照（用于识别元素层次）
- 使用 `browser.screenshot({ fullPage: true })` 截取完整页面截图
- 保存截图到功能目录（由主命令处理）

#### 4.2 提取 HTML 结构（可选）

**操作**: 使用 `browser.evaluate(() => document.documentElement.outerHTML)` 提取完整 HTML 源码

**使用场景**:
- 需要分析DOM结构时
- 需要提取隐藏元素时
- 需要分析页面类型时

---

### 阶段 5: 错误处理

#### 5.1 统一错误类型

```typescript
type ErrorType = 
  | 'INVALID_URL'        // URL格式不正确
  | 'LOGIN_REQUIRED'     // 需要登录
  | 'PAGE_NOT_FOUND'     // 页面不存在
  | 'NETWORK_ERROR'      // 网络错误
  | 'CONTENT_EXTRACT_FAILED'  // 内容提取失败
  | 'UNKNOWN'            // 未知错误
```

#### 5.2 错误处理流程

1. **捕获错误**: 在关键步骤使用 try-catch 捕获异常
2. **错误分类**: 根据错误类型进行分类
3. **返回错误**: 返回统一的错误格式

#### 5.3 错误返回格式

```javascript
{
  success: false,
  error: {
    type: ErrorType,
    message: string,  // 用户友好的错误信息
    url: string,      // 出错的URL
    details?: any     // 可选的错误详情
  }
}
```

## 🔧 通用辅助函数

### validateUrl(url: string, options?: ValidationOptions)

验证URL格式，支持特定格式检测。

**参数**:
- `url`: 要验证的URL
- `options`: 验证选项
  - `requiredKeywords?: string[]`: 必须包含的关键词（不区分大小写）
  - `allowedDomains?: string[]`: 允许的域名列表

**返回**: `{ valid: boolean, error?: string }`

### detectLoginRequired(snapshot: Snapshot): boolean

检测页面是否需要登录。

**参数**:
- `snapshot`: 页面快照

**返回**: `boolean` - 是否需要登录

### handleLogin(loginStrategy: 'wait' | 'prompt' | 'skip'): Promise<boolean>

处理登录流程。

**参数**:
- `loginStrategy`: 登录策略
  - `'wait'`: 等待用户手动登录
  - `'prompt'`: 提示用户提供替代方案
  - `'skip'`: 跳过登录（通常不需要）

**返回**: `Promise<boolean>` - 是否成功处理登录

### createErrorResponse(type: ErrorType, message: string, url: string, details?: any)

创建统一的错误响应。

**参数**:
- `type`: 错误类型
- `message`: 错误信息
- `url`: 出错的URL
- `details`: 可选的错误详情

**返回**: 错误响应对象

### createSuccessResponse(featureDescription: string, sourceInfo: SourceInfo)

创建统一的成功响应。

**参数**:
- `featureDescription`: 功能描述文本
- `sourceInfo`: 来源信息

**返回**: 成功响应对象
