# 环境与依赖检查模块

---
**模块名称**: 环境与依赖检查  
**版本**: 1.0  
**创建时间**: 2025-01-XX  
**用途**: 检查基础环境（Node.js、包管理器、Python、Chrome）和项目依赖，并自动修复/安装  
**调用方**: `设计预览.md` 命令、`前端设计.md` 命令  
---

## 📋 模块概述

本模块负责检查预览所需的基础环境（Node.js、npm/yarn/pnpm、Python、Chrome浏览器）和项目依赖，并在缺失时提供安装指南或自动安装。

---

## 🔄 执行流程

### 阶段1：基础环境检查与自动修复

#### 1.1 Node.js环境检查与修复

**检查步骤**：

1. **检查Node.js是否安装**：
   ```bash
   node --version
   ```
   - ✅ 如果成功返回版本号 → Node.js已安装
   - ❌ 如果命令不存在 → Node.js未安装

2. **如果Node.js未安装**：
   - **Mac系统**：
     - 提示用户安装方式：
       ```bash
       # 使用Homebrew安装（推荐）
       brew install node
       
       # 或从官网下载安装包
       # https://nodejs.org/
       ```
     - 提供详细安装指南链接
     - **等待用户安装完成后继续**（询问用户是否已安装）

   - **Windows系统**：
     - 提示用户从官网下载安装包：https://nodejs.org/
     - 提供安装步骤说明
     - **等待用户安装完成后继续**

   - **Linux系统**：
     - 提供安装命令：
       ```bash
       # Ubuntu/Debian
       sudo apt-get update
       sudo apt-get install nodejs npm
       
       # CentOS/RHEL
       sudo yum install nodejs npm
       ```

3. **检查npm是否可用**：
   ```bash
   npm --version
   ```
   - 如果不可用，尝试重新安装Node.js（npm通常随Node.js一起安装）

#### 1.2 包管理器检查与修复

**检查步骤**：

1. **检查可用的包管理器**（按优先级）：
   - `yarn --version` → 检查Yarn
   - `pnpm --version` → 检查pnpm
   - `npm --version` → 检查npm（应该已安装）

2. **如果项目需要特定包管理器但未安装**：

   **安装Yarn**：
   ```bash
   # 全局安装Yarn
   npm install -g yarn
   ```

   **安装pnpm**：
   ```bash
   # 使用npm安装
   npm install -g pnpm
   
   # 或使用独立安装脚本
   curl -fsSL https://get.pnpm.io/install.sh | sh -
   ```

3. **验证安装**：
   - 重新检查包管理器命令是否可用
   - 如果仍不可用，提示用户手动安装

#### 1.3 Python环境检查（HTML原型需要）

**检查步骤**：

1. **检查Python是否可用**（用于启动静态HTTP服务器）：
   ```bash
   python3 --version
   # 或
   python --version
   ```

2. **如果Python未安装**：
   - **Mac系统**：
     - Mac通常自带Python，如果不可用：
       ```bash
       brew install python3
       ```

   - **Windows系统**：
     - 提示从官网下载：https://www.python.org/downloads/
     - 安装时勾选"Add Python to PATH"

   - **Linux系统**：
     ```bash
     # Ubuntu/Debian
     sudo apt-get install python3
     
     # CentOS/RHEL
     sudo yum install python3
     ```

3. **如果Python不可用且Node.js可用**：
   - 使用Node.js的`http-server`作为备选方案（不需要Python）

#### 1.4 Chrome浏览器检查与安装提示

**检查步骤**：

1. **检查Chrome是否安装**（按系统）：

   **Mac系统**：
   ```bash
   test -d "/Applications/Google Chrome.app" && echo "Chrome已安装" || echo "Chrome未安装"
   ```

   **Windows系统**：
   ```bash
   # 检查常见安装路径
   test -f "C:\Program Files\Google\Chrome\Application\chrome.exe" || test -f "C:\Program Files (x86)\Google\Chrome\Application\chrome.exe"
   ```

   **Linux系统**：
   ```bash
   which google-chrome || which chromium-browser
   ```

2. **如果Chrome未安装**：
   - 提供下载链接：https://www.google.com/chrome/
   - 提供安装说明
   - **备选方案**：使用系统默认浏览器（但提示Chrome体验更好）

3. **验证Chrome可执行**：
   - 尝试执行Chrome命令，确认可以正常启动


#### 1.5 Live Server扩展检查与自动安装（HTML预览需要）

**检查步骤**：

1. **检查Live Server扩展是否已安装**：

   **方法1：检查扩展目录**（VS Code/Cursor）：
   
   **Mac系统**：
   ```bash
   # VS Code
   ls -d "$HOME/.vscode/extensions/ritwickdey.liveserver-"* 2>/dev/null && echo "Live Server已安装" || echo "Live Server未安装"
   
   # Cursor
   ls -d "$HOME/.cursor/extensions/ritwickdey.liveserver-"* 2>/dev/null && echo "Live Server已安装" || echo "Live Server未安装"
   ```

   **Windows系统**：
   ```bash
   # VS Code
   if exist "%USERPROFILE%\.vscode\extensions\ritwickdey.liveserver-*" (echo Live Server已安装) else (echo Live Server未安装)
   
   # Cursor
   if exist "%USERPROFILE%\.cursor\extensions\ritwickdey.liveserver-*" (echo Live Server已安装) else (echo Live Server未安装)
   ```

   **Linux系统**：
   ```bash
   # VS Code
   ls -d "$HOME/.vscode/extensions/ritwickdey.liveserver-"* 2>/dev/null && echo "Live Server已安装" || echo "Live Server未安装"
   
   # Cursor
   ls -d "$HOME/.cursor/extensions/ritwickdey.liveserver-"* 2>/dev/null && echo "Live Server已安装" || echo "Live Server未安装"
   ```

   **方法2：通过命令行检查扩展列表**：
   ```bash
   # VS Code
   code --list-extensions 2>/dev/null | grep -i "ritwickdey.liveserver" && echo "Live Server已安装" || echo "Live Server未安装"
   
   # Cursor（如果支持）
   cursor --list-extensions 2>/dev/null | grep -i "ritwickdey.liveserver" && echo "Live Server已安装" || echo "Live Server未安装"
   ```

2. **如果Live Server未安装，自动安装**：

   **检测当前使用的编辑器**：
   - 检查环境变量或进程，判断是VS Code还是Cursor
   - 如果无法确定，尝试两种方式

   **安装Live Server扩展**：

   **VS Code**：
   ```bash
   code --install-extension ritwickdey.LiveServer
   ```

   **Cursor**：
   ```bash
   # Cursor可能使用相同的命令，或需要不同的方式
   cursor --install-extension ritwickdey.LiveServer 2>/dev/null
   # 如果上述命令失败，提示用户手动安装：
   # 1. 打开Cursor
   # 2. 点击扩展图标
   # 3. 搜索"Live Server"
   # 4. 安装"Live Server"（作者：Ritwick Dey）
   ```

3. **验证安装结果**：
   - 重新检查扩展目录或扩展列表
   - 如果安装成功，继续后续流程
   - 如果安装失败，使用备选方案（Python HTTP服务器或Node.js http-server）

4. **启动HTML文件预览**（当需要预览HTML文件时）：

   **优先使用Live Server**：
   - 如果Live Server已安装，尝试通过命令行启动：
     ```bash
     # 方法1：使用code/cursor命令打开文件（会自动使用Live Server）
     code "$HTML_FILE_PATH"
     # 或
     cursor "$HTML_FILE_PATH"
     ```
   - 如果无法通过命令行启动Live Server，提示用户：
     - 在编辑器中右键点击HTML文件
     - 选择"Open with Live Server"

   **备选方案1：Python HTTP服务器**（如果Python可用）：
   ```bash
   # 在HTML文件所在目录启动
   cd "$(dirname "$HTML_FILE_PATH")"
   python3 -m http.server 8000 &
   # 然后在浏览器中打开：http://localhost:8000/文件名.html
   open "http://localhost:8000/$(basename "$HTML_FILE_PATH")"
   ```

   **备选方案2：Node.js http-server**（如果Node.js可用）：
   ```bash
   # 安装http-server（如果未安装）
   npm install -g http-server 2>/dev/null || true
   
   # 在HTML文件所在目录启动
   cd "$(dirname "$HTML_FILE_PATH")"
   http-server -p 8000 -c-1 &
   # 然后在浏览器中打开：http://localhost:8000/文件名.html
   open "http://localhost:8000/$(basename "$HTML_FILE_PATH")"
   ```

   **备选方案3：直接打开文件**（最后手段）：
   ```bash
   # Mac
   open "$HTML_FILE_PATH"
   
   # Windows
   start "$HTML_FILE_PATH"
   
   # Linux
   xdg-open "$HTML_FILE_PATH"
   ```
   ⚠️ 注意：使用file://协议打开可能遇到CORS限制，建议使用HTTP服务器

### 阶段2：项目依赖检查与自动安装

#### 2.1 检查项目依赖（前端项目）

**检查步骤**：

1. **检查`node_modules/`目录**：
   - 如果存在且不为空 → 依赖可能已安装
   - 如果不存在或为空 → 需要安装依赖

2. **检查`package.json`中的依赖**：
   - 读取`package.json`文件
   - 检查`dependencies`和`devDependencies`
   - 如果存在依赖但`node_modules/`为空 → 需要安装

3. **检查锁文件**：
   - `yarn.lock` → 使用Yarn安装
   - `pnpm-lock.yaml` → 使用pnpm安装
   - `package-lock.json` → 使用npm安装
   - 无锁文件 → 使用npm安装（默认）

#### 2.2 自动安装依赖

**安装策略**：

1. **根据锁文件选择包管理器**：
   ```bash
   # 如果存在yarn.lock
   yarn install
   
   # 如果存在pnpm-lock.yaml
   pnpm install
   
   # 否则使用npm
   npm install
   ```

2. **安装过程监控**：
   - 监控安装输出
   - 检测安装错误：
     - `npm ERR!` → npm错误
     - `yarn error` → Yarn错误
     - `ENOENT` → 文件不存在
     - `EACCES` → 权限错误
     - `network timeout` → 网络超时

3. **错误处理**：

   **权限错误（EACCES）**：
   - **Mac/Linux**：提示使用sudo（不推荐）或修复npm权限
     ```bash
     # 修复npm权限（推荐）
     mkdir ~/.npm-global
     npm config set prefix '~/.npm-global'
     export PATH=~/.npm-global/bin:$PATH
     ```
   - **Windows**：以管理员身份运行

   **网络超时**：
   - 提示切换npm镜像源：
     ```bash
     # 使用淘宝镜像
     npm config set registry https://registry.npmmirror.com
     
     # 或使用cnpm
     npm install -g cnpm --registry=https://registry.npmmirror.com
     cnpm install
     ```

   **依赖冲突**：
   - 尝试使用`--legacy-peer-deps`（npm）或`--ignore-peer-deps`（yarn）
   - 如果仍失败，显示错误详情，需要用户手动解决

4. **验证安装结果**：
   - 检查`node_modules/`目录是否已创建
   - 检查关键依赖是否已安装
   - 如果安装失败，重试或提示用户

#### 2.3 缺失依赖的自动检测与安装

**运行时依赖检测**：

1. **启动服务器时检测**：
   - 监控启动输出
   - 检测错误：`Cannot find module 'xxx'`、`Module not found: 'xxx'`

2. **提取缺失的包名**：
   - 从错误信息中提取包名
   - 检查是`dependencies`还是`devDependencies`

3. **自动安装缺失的依赖**：
   ```bash
   # 根据包管理器安装
   npm install <package-name>
   # 或
   yarn add <package-name>
   # 或
   pnpm add <package-name>
   ```

4. **重新启动服务器**：
   - 安装完成后，自动重新启动服务器
   - 继续监控错误

## 📝 模块使用说明

在主命令中，通过以下方式调用本模块：

```markdown
### 阶段1和阶段2：环境与依赖检查

**执行模块**：`modules/环境与依赖检查.md`

请读取并执行该模块中的所有检查步骤，包括：
- 基础环境检查（Node.js、包管理器、Python、Chrome）
- 项目依赖检查与自动安装
- 运行时缺失依赖的自动检测与安装

执行完成后，继续执行主命令的后续阶段。
```
