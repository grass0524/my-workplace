## 用户输入

```text
$ARGUMENTS
```

在继续之前，你**必须**考虑用户输入（如果不为空）。

## 概述

本命令用于预览设计完的原型或前端设计页面。命令会自动识别预览目标类型，启动相应的本地服务器，使用Chrome浏览器打开页面，并持续检测和修复错误（编译错误、端口冲突、依赖缺失、环境问题等），直到用户能够成功预览页面。

**核心特性**：
- ✅ **自动识别预览目标**：支持HTML原型文件和前端项目（Vue/React等）
- ✅ **智能启动服务器**：根据项目类型自动选择启动方式（静态服务器/开发服务器）
- ✅ **Chrome浏览器优先**：优先使用Chrome浏览器打开页面
- ✅ **错误自动检测**：持续监控编译错误、端口冲突、依赖缺失、环境问题等
- ✅ **自动修复错误**：自动修复常见错误，包括依赖安装、环境配置等
- ✅ **持续监控**：保持服务器运行，监控错误状态

## 执行流程

### 阶段0：输入识别与目标定位

#### 0.1 识别用户输入

**输入类型识别**（按优先级）：

1. **指定路径**：用户提供了文件或目录路径
   - HTML文件路径（`.html`结尾）→ **HTML原型文件**
   - 目录路径且包含`package.json` → **前端项目**
   - 目录路径且包含`.html`文件 → **HTML原型目录**

2. **未指定路径**：用户未提供路径
   - 搜索`docs/prototypes/`目录下的最新版本原型
   - 搜索项目根目录下的前端项目（查找`package.json`）
   - 如果找到多个，列出让用户选择

#### 0.2 确定预览目标类型

**目标类型判断**：

| 特征 | 类型 | 处理方式 |
|------|------|---------|
| 包含`package.json` | **前端项目** | 启动开发服务器（npm/yarn/pnpm） |
| 包含`.html`文件且无`package.json` | **HTML原型** | 启动静态HTTP服务器 |
| 单个`.html`文件 | **HTML文件** | 启动静态HTTP服务器 |

#### 0.3 验证预览目标

**验证项**：
- ✅ 目标文件/目录存在
- ✅ 如果是前端项目，检查`package.json`是否存在
- ✅ 如果是HTML原型，检查至少有一个`.html`文件
- ✅ 检查必要的依赖和工具是否已安装

### 阶段3：启动本地服务器

#### 3.1 前端项目 - 启动开发服务器

**步骤1：读取项目端口配置**

根据项目类型读取配置文件中的端口设置：

1. **Vite项目**（存在`vite.config.ts`或`vite.config.js`）：
   - 读取配置文件中的`server.port`配置
   - 如果未配置，Vite默认使用`5173`
   - 示例：`vite.config.ts`中`server: { port: 8090 }` → 使用端口`8090`

2. **Vue CLI项目**（存在`vue.config.js`）：
   - 读取`devServer.port`配置
   - 如果未配置，Vue CLI默认使用`8080`

3. **React项目**（Create React App）：
   - 检查环境变量`PORT`或`package.json`中的端口配置
   - 默认使用`3000`

4. **其他项目**：
   - 尝试从配置文件读取端口配置
   - 如果无法确定，使用项目默认端口

**步骤2：检查端口占用**

在启动服务器前，检查配置的端口是否被占用：

1. **检查端口占用**：
   ```bash
   # Mac/Linux
   lsof -i :端口号
   
   # Windows
   netstat -ano | findstr :端口号
   ```

2. **如果端口被占用**：
   - 显示占用端口的进程信息
   - 提示用户：
     ```
     ⚠️ 端口 {端口号} 已被占用
     
     占用进程：{进程信息}
     
     请选择处理方式：
     1. 关闭占用端口的进程（推荐）
     2. 使用其他端口（将修改配置文件）
     3. 取消启动
     ```
   - 等待用户选择处理方式

3. **如果端口未被占用**：
   - 继续执行启动流程

**步骤3：识别启动命令**

1. **检查`package.json`中的scripts**：
   - 优先查找：`dev`、`serve`、`start`、`develop`
   - 如果找到，使用对应的命令

2. **根据包管理器选择命令**：
   ```bash
   # Yarn项目
   yarn dev 或 yarn serve 或 yarn start
   
   # pnpm项目
   pnpm dev 或 pnpm serve 或 pnpm start
   
   # npm项目
   npm run dev 或 npm run serve 或 npm start
   ```

**步骤4：启动服务器**

1. **在后台启动开发服务器**：
   - 使用步骤3识别的启动命令
   - 捕获输出日志，用于错误检测

2. **验证端口使用**：
   - 等待服务器启动完成（检测到"Local:"、"http://"或"ready"输出）
   - 从输出中提取实际使用的端口号
   - **验证端口一致性**：
     - 如果实际端口与配置端口不一致：
       - 检查是否因为端口被占用而自动切换
       - 如果是，提示用户实际使用的端口
       - 记录实际端口用于后续浏览器打开
     - 如果端口一致：
       - 确认服务器已成功启动在配置的端口上

#### 3.2 HTML原型 - 启动静态HTTP服务器

**服务器选择**（按优先级）：

1. **Python HTTP服务器**（如果Python可用）：
   ```bash
   # Python 3（优先）
   python3 -m http.server 8000
   
   # Python 2（备选）
   python -m SimpleHTTPServer 8000
   ```

2. **Node.js HTTP服务器**（如果Node.js可用）：
   ```bash
   # 使用npx（无需全局安装）
   npx http-server -p 8000 -c-1
   
   # 如果http-server未安装，先安装
   npm install -g http-server
   http-server -p 8000 -c-1
   ```

3. **Live Server插件**（如果VS Code/Cursor已安装）：
   - 检查是否安装了Live Server扩展
   - 如果已安装，可以使用扩展启动

**端口选择**：
- 默认端口：`8000`
- 如果端口被占用，自动尝试：`8001`、`8002`、`8080`、`3000`、`5173`、`5174`
- 记录实际使用的端口号

### 阶段4和阶段6：错误检测与监控

**执行模块**：`.cursor/modules/错误检测与监控.md`

请读取并执行该模块中的所有步骤，包括：
- **端口冲突检测**：检测配置端口与实际使用端口是否一致，如果不一致则提示用户
- **编译错误检测**：检测编译过程中的错误信息
- **依赖缺失检测**：检测运行时缺失的依赖包
- **配置错误检测**：检测项目配置文件中的错误
- **按优先级自动修复**：自动修复可修复的错误（如安装缺失依赖）
- **持续监控**：持续监控服务器运行状态
- **生成预览状态报告**：生成包含端口信息、错误状态等的预览报告

**⚠️ 重要**：如果检测到端口不一致（配置端口与实际端口不同），必须：
1. 显示警告信息，说明实际使用的端口
2. 记录实际端口用于浏览器打开
3. 如果是因为端口被占用导致的切换，提示用户处理占用进程

执行完成后，继续执行主命令的后续阶段。

### 阶段5：打开浏览器预览

#### 5.1 确定访问URL

**URL确定规则**：

- **前端项目**：
  - **优先使用配置的端口**：使用步骤3.1中读取的项目配置文件中的端口
  - **备选方案**：从服务器输出中提取URL（通常为`http://localhost:端口号`）
  - **端口优先级**：
    1. 配置文件中的端口（如`vite.config.ts`中的`server.port: 8090`）
    2. 服务器输出中提取的实际端口
    3. 项目默认端口（Vite: 5173, Vue CLI: 8080, React: 3000）
  - **URL格式**：`http://localhost:{端口号}` 或 `http://127.0.0.1:{端口号}`
  - **端口验证**：如果实际端口与配置端口不一致，在URL中显示实际端口，并提示用户

- **HTML原型**：
  - 使用步骤3.2中确定的端口号
  - 如果是单个HTML文件：`http://localhost:端口号/文件名.html`
  - 如果是目录：
    - 优先：`http://localhost:端口号/index.html`
    - 备选：`http://localhost:端口号/`（列出目录）

#### 5.2 使用Chrome浏览器打开

**打开命令**：

- **Mac**：
  ```bash
  open -a "Google Chrome" "http://localhost:端口号"
  ```

- **Windows**：
  ```bash
  start chrome "http://localhost:端口号"
  # 或使用完整路径
  "C:\Program Files\Google\Chrome\Application\chrome.exe" "http://localhost:端口号"
  ```

- **Linux**：
  ```bash
  google-chrome "http://localhost:端口号" 2>/dev/null || \
  chromium-browser "http://localhost:端口号" 2>/dev/null || \
  chromium "http://localhost:端口号"
  ```

**备选方案**（如果Chrome不可用）：
- **Mac**：
  ```bash
  open "http://localhost:端口号"  # 使用系统默认浏览器
  ```

- **Windows**：
  ```bash
  start "http://localhost:端口号"  # 使用系统默认浏览器
  ```

- **Linux**：
  ```bash
  xdg-open "http://localhost:端口号"  # 使用系统默认浏览器
  ```

#### 5.3 验证页面可访问性

**验证步骤**：
1. 等待浏览器打开（2-3秒）
2. 检查服务器日志，确认有HTTP请求（200状态码）
3. 如果检测到404或其他错误，进入错误修复流程

**页面加载错误检测**：
- **404错误**：
  - 检查文件路径是否正确
  - 检查`index.html`是否存在
  - 修复路径或创建缺失文件

- **资源加载错误**（CSS/JS/图片）：
  - 检查资源路径是否正确（相对路径/绝对路径）
  - 检查资源文件是否存在
  - 修复资源路径

- **CORS错误**：
  - 如果是静态服务器，通常不会有CORS问题
  - 如果是开发服务器，检查CORS配置

## 命令参数说明

```
/设计预览 [选项] [路径]

选项:
  --port=<port>          指定服务器端口（默认：自动选择）
  --browser=<browser>    指定浏览器（chrome/firefox/safari/default）
  --no-auto-fix          禁用自动修复（仅检测错误，不自动修复）
  --no-install           跳过依赖安装（假设依赖已安装）
  --watch                持续监控模式（默认启用）

路径:
  如果未指定路径，自动搜索预览目标

示例:
  /设计预览                                    # 自动搜索并预览
  /设计预览 docs/prototypes/user/v1/          # 预览指定原型目录
  /设计预览 ./my-project                      # 预览指定前端项目
  /设计预览 index.html                        # 预览指定HTML文件
  /设计预览 --port=8080                       # 指定端口
  /设计预览 --browser=chrome                  # 指定浏览器
  /设计预览 --no-auto-fix                    # 仅检测错误，不自动修复
```

## 注意事项

1. **服务器持续运行**：
   - 预览启动后，服务器会持续运行
   - 需要手动停止（在终端按Ctrl+C）
   - 停止后会自动清理进程

2. **端口占用**：
   - 如果指定端口被占用，会自动切换到其他端口
   - 建议使用默认的自动端口选择

3. **浏览器选择**：
   - 优先使用Chrome浏览器
   - 如果Chrome不可用，使用系统默认浏览器
   - 可以通过`--browser`参数指定浏览器

4. **依赖安装时间**：
   - 大型项目的依赖安装可能需要几分钟
   - 首次安装会显示进度
   - 如果网络较慢，建议切换npm镜像源

5. **自动修复限制**：
   - 只能自动修复常见错误
   - 复杂错误需要用户手动修复
   - 修复后如果仍有问题，会显示详细错误信息

6. **环境要求**：
   - 前端项目需要Node.js环境
   - HTML原型可以使用Python或Node.js的HTTP服务器
   - Chrome浏览器推荐但不强制
