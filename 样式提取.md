## 用户输入

```text
$ARGUMENTS
```

在继续之前，你**必须**考虑用户输入（如果不为空）。

## 概述

**命令名称**: `/样式提取`  
**命令用途**: 从网页或图片中提取前端样式规范，生成标准化的设计规范文档  
**适用场景**: 
- 需要学习参考其他产品的设计规范
- 需要为现有项目反向提取设计规范
- 需要分析竞品的设计体系
- 需要建立团队统一的设计语言

## 命令格式

```
/样式提取 <来源> [选项]
```

### 参数说明

| 参数 | 必填 | 类型 | 说明 | 示例 |
|------|------|------|------|------|
| `<来源>` | ✅ | string/array | 网页URL或图片路径，支持多个 | `https://example.com` |
| `--name` | ❌ | string | 生成的规范文档名称 | `--name kubeease` |
| `--output` | ❌ | string | 输出目录路径（默认：`rules/company/Product-Design/`） | `--output rules/company/Product-Design/` |
| `--type` | ❌ | string | 提取类型：`full`(完整) / `colors`(仅颜色) / `layout`(仅布局) / `components`(仅组件) | `--type full` |
| `--pages` | ❌ | number | 提取的页面数量（多页面时） | `--pages 5` |
| `--screenshot` | ❌ | boolean | 是否保存截图 | `--screenshot` |
| `--format` | ❌ | string | 输出格式：`standard`(标准目录结构) / `simple`(单文件) | `--format standard` |
| `--tokens` | ❌ | string | 设计变量格式：`both`(CSS+Less) / `css`(仅CSS) / `less`(仅Less) | `--tokens both` |

## 使用示例

### 示例：提取完整样式规范

```
/样式提取
http://kubeease-rel.kubeease.cn/tenants/bdtech/overview
http://kubeease-rel.kubeease.cn/tenants/bdtech/projectlist
--name kubeease
--format standard
--tokens both
--screenshot
--pages 2
```

**说明**: 提取 Kubeease 平台样式，生成完整规范目录，包含原型设计和前端开发所需的所有文件。文件将默认保存到 `rules/company/Product-Design/kubeease-design-specs/` 目录。

## 执行流程

### 阶段 1: 准备与验证

1. 解析命令参数
2. 验证来源有效性（URL 可访问或图片文件存在）
3. 确认输出路径（如未指定 `--output`，默认使用 `rules/company/Product-Design/`）
4. 初始化浏览器或图片分析工具

### 阶段 2: 数据采集

**网页样式采集**：
1. 导航到目标页面
2. 等待页面完全加载
3. 获取页面快照和截图
4. 提取关键元素样式

**图片样式采集**：
1. 读取图片文件
2. 分析图片尺寸和布局
3. 识别颜色值
4. 识别文字和字体信息（OCR）
5. 识别组件结构

### 阶段 3: 样式分析

- **颜色分析**：提取主色调、功能色、中性色体系
- **布局分析**：识别页面结构、测量尺寸、分析栅格系统、测量间距规律
- **字体分析**：识别字体家族、提取字号体系、分析行高规律、识别字重使用
- **组件分析**：识别按钮、输入框、卡片、表格等组件样式
- **交互分析**：悬停效果、过渡动画时长、阴影和圆角规律

### 阶段 4: 规范生成

**文档结构**（标准目录结构）：
```
rules/company/Product-Design/{name}-design-specs/
├── README.md                    # 规范索引文件
├── design-system.md            # 核心设计系统（布局、颜色、间距、字体）
├── component-specs.md          # 组件详细规范（按钮、表单、表格、弹窗等）
├── design-tokens.css           # 设计变量（CSS 版本，用于原型设计）
├── design-tokens.less          # 设计变量（Less 版本，用于前端开发）
├── prototype-guide.md          # 原型设计指南
├── frontend-guide.md           # 前端开发指南
└── screenshots/                 # 截图目录
```

**注意**：所有提取的样式规范文件夹默认保存在 `rules/company/Product-Design/` 目录下。如果指定了 `--output` 参数，则保存到指定路径。

**核心文档内容**：
- **README.md**：规范说明、规范文件清单、快速开始指南、规范使用流程
- **design-system.md**：布局系统、颜色系统、间距系统、字体规范、圆角规范、过渡动画、阴影规范
- **component-specs.md**：按钮、表单、表格、弹窗、筛选器、状态徽章、分页等组件规范
- **design-tokens.css/less**：设计变量文件，包含颜色、间距、圆角、字体、组件尺寸、阴影、过渡、布局等变量

### 阶段 5: 输出与保存

1. 确定最终输出路径：
   - 如果指定了 `--output` 参数，使用指定路径
   - 如果未指定，默认保存到 `rules/company/Product-Design/{name}-design-specs/`
2. 创建目录结构
3. 生成并保存所有规范文档文件
4. 如启用了 `--screenshot`，保存截图到 `screenshots/` 目录

## 提取内容清单

### 必提取内容

- 主色调、功能色、中性色体系
- 布局结构、间距体系、圆角规范
- 字号体系、字重规范、行高规范

### 建议提取内容

- 按钮规范、输入框规范、表格样式、卡片样式
- 导航样式、表单规范、模态框、标签徽章、分页器

### 高级提取内容

- 阴影体系、动画规范、响应式断点、深色主题、图表样式、图标规范、加载状态、空状态

## 质量检查清单

提取完成后，检查以下核心项：

- **颜色系统**：主色调、功能色、中性色体系完整，颜色值格式统一
- **布局系统**：页面结构清晰，尺寸数值准确，间距规律明确
- **字体系统**：字体家族明确，字号体系完整，行高合理
- **组件规范**：至少包含5个核心组件，每个组件包含尺寸、样式、状态
- **文档质量**：文档结构完整，表格格式正确，代码示例可用，截图清晰

## 注意事项

1. **输出路径**：提取的样式规范文件夹默认保存在 `rules/company/Product-Design/` 目录下，可通过 `--output` 参数自定义
2. **提取目标选择**：推荐提取成熟的设计系统、优秀的商业产品、内部已上线的成熟产品
3. **多页面提取策略**：建议提取首页、列表页、表单页、详情页、设置页等典型页面
4. **验证与核对**：提取完成后对比原页面和提取的规范，使用提取的值在设计工具中验证
5. **持续维护**：定期更新规范文档，记录设计演变历史，同步最新的设计变更
