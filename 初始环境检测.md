## 用户输入

```text
$ARGUMENTS
```

在继续之前，你**必须**考虑用户输入（如果不为空）。

## 概述

本命令用于检测Windows和Mac操作系统上使用Cursor工具进行开发所需的基础环境，包括Git、Node.js、包管理器、Python、Chrome浏览器等。检测到缺失的软件时，会自动执行安装或提供详细的安装指南。支持使用国内镜像源加速下载和安装。

**核心特性**：
- ✅ **跨平台支持**：支持Windows 10/11和macOS 10.15+系统
- ✅ **自动检测**：检测所有必需的开发环境和工具
- ✅ **智能安装**：自动安装缺失的软件，支持多种安装方式
- ✅ **国内加速**：自动检测网络环境，可选使用国内镜像源
- ✅ **版本验证**：检测已安装软件的版本是否符合要求
- ✅ **详细报告**：生成环境检测报告，显示所有检测结果

**检测项目**：
1. 操作系统版本和架构
2. Git版本控制工具
3. Node.js运行环境（≥16.0）
4. npm/yarn/pnpm包管理器
5. Python环境（≥3.7）
6. Chrome/Edge浏览器
7. Cursor扩展（Live Server等）
8. 网络连接和镜像源可用性

## 执行流程

### 阶段0：系统信息检测

#### 0.1 检测操作系统

**检测步骤**：

1. **识别操作系统类型**：
   ```bash
   # 通过环境变量识别
   # Windows: %OS% = Windows_NT
   # Mac: uname -s = Darwin
   ```

2. **检测系统版本**：
   
   **Windows**：
   ```powershell
   # PowerShell命令
   systeminfo | findstr /B /C:"OS Name" /C:"OS Version"
   # 或
   [System.Environment]::OSVersion.Version
   ```
   
   **Mac**：
   ```bash
   # 获取macOS版本
   sw_vers -productVersion
   # 输出示例：12.6.0 (Monterey)
   ```

3. **检测系统架构**：
   
   **Windows**：
   ```powershell
   # 检测是x64还是ARM64
   $env:PROCESSOR_ARCHITECTURE
   # 或
   systeminfo | findstr /C:"System Type"
   ```
   
   **Mac**：
   ```bash
   # 检测是Intel还是Apple Silicon
   uname -m
   # x86_64 = Intel
   # arm64 = Apple Silicon (M1/M2/M3)
   ```

4. **显示系统信息**：
   ```
   🖥️ 系统信息
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   操作系统: {Windows 11 / macOS Monterey}
   版本号: {版本}
   架构: {x64 / ARM64 / Apple Silicon}
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ```

#### 0.2 检测网络环境

**检测步骤**：

1. **测试国际网络连接**：
   ```bash
   # 测试访问GitHub
   ping -c 1 github.com 2>&1
   # 或
   curl -I --connect-timeout 3 https://github.com 2>&1
   ```

2. **测试国内镜像源**：
   ```bash
   # 测试访问淘宝npm镜像
   curl -I --connect-timeout 3 https://registry.npmmirror.com 2>&1
   ```

3. **确定网络策略**：
   - ✅ 国际网络可用 → 优先使用官方源，备选国内源
   - ⚠️ 国际网络不稳定 → 推荐使用国内源
   - ❌ 国际网络不可用 → 强制使用国内源

4. **显示网络状态**：
   ```
   🌐 网络连接检测
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   国际网络: {可用 / 不稳定 / 不可用}
   国内镜像: {可用 / 不可用}
   建议策略: {官方源 / 国内镜像源}
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ```

### 阶段1：核心开发工具检测与安装

#### 1.1 Git版本控制工具

**检测步骤**：

1. **检查Git是否安装**：
   ```bash
   git --version
   ```
   - ✅ 输出版本号（如 `git version 2.40.0`）→ Git已安装
   - ❌ 命令不存在 → Git未安装

2. **如果Git未安装，执行自动安装**：

   **Windows系统**：
   
   **方式1：使用Winget（Windows 10/11推荐）**：
   ```powershell
   # 检查winget是否可用
   winget --version
   
   # 使用winget安装Git
   winget install --id Git.Git -e --source winget
   ```
   
   **方式2：使用Chocolatey**：
   ```powershell
   # 检查chocolatey是否安装
   choco --version
   
   # 如果未安装，先安装Chocolatey
   Set-ExecutionPolicy Bypass -Scope Process -Force
   [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
   iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
   
   # 使用Chocolatey安装Git
   choco install git -y
   ```
   
   **方式3：手动下载安装**：
   ```
   📥 请从以下地址下载Git安装包：
   
   官方下载：https://git-scm.com/download/win
   国内镜像：https://npm.taobao.org/mirrors/git-for-windows/
   
   安装提示：
   1. 下载适合您系统架构的安装包（64位/32位）
   2. 运行安装程序，建议使用默认设置
   3. 安装完成后重启终端或IDE
   ```

   **Mac系统**：
   
   **方式1：使用Homebrew（推荐）**：
   ```bash
   # 检查Homebrew是否安装
   brew --version
   
   # 如果未安装，先安装Homebrew
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   
   # 国内镜像（如果官方源不可用）
   /bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"
   
   # 使用Homebrew安装Git
   brew install git
   ```
   
   **方式2：使用Xcode Command Line Tools**：
   ```bash
   # 安装Xcode命令行工具（包含Git）
   xcode-select --install
   ```
   
   **方式3：手动下载安装**：
   ```
   📥 请从以下地址下载Git安装包：
   
   官方下载：https://git-scm.com/download/mac
   ```

3. **验证安装结果**：
   ```bash
   # 重新检查Git版本
   git --version
   
   # 检查Git配置
   git config --list
   ```

4. **配置Git基本信息**（如果是首次安装）：
   ```bash
   # 设置用户名
   git config --global user.name "Your Name"
   
   # 设置邮箱
   git config --global user.email "your.email@example.com"
   
   # 配置默认编辑器
   git config --global core.editor "code --wait"
   
   # 配置行尾符处理（Windows）
   git config --global core.autocrlf true
   
   # 配置行尾符处理（Mac/Linux）
   git config --global core.autocrlf input
   ```

5. **配置Git加速（可选）**：
   ```bash
   # 使用国内镜像加速Git克隆
   git config --global url."https://gitclone.com/github.com/".insteadOf "https://github.com/"
   
   # 或使用FastGit镜像
   git config --global url."https://hub.fastgit.xyz/".insteadOf "https://github.com/"
   ```

#### 1.2 Node.js运行环境

**检测步骤**：

1. **检查Node.js是否安装及版本**：
   ```bash
   node --version
   ```
   - ✅ 版本 ≥ v16.0.0 → Node.js已安装且版本符合要求
   - ⚠️ 版本 < v16.0.0 → Node.js版本过低，需要升级
   - ❌ 命令不存在 → Node.js未安装

2. **如果Node.js未安装或版本过低，执行安装/升级**：

   **Windows系统**：
   
   **方式1：使用Winget（推荐）**：
   ```powershell
   # 安装LTS版本
   winget install OpenJS.NodeJS.LTS
   
   # 或安装最新版本
   winget install OpenJS.NodeJS
   ```
   
   **方式2：使用Chocolatey**：
   ```powershell
   # 安装LTS版本
   choco install nodejs-lts -y
   
   # 或安装最新版本
   choco install nodejs -y
   ```
   
   **方式3：使用nvm-windows（版本管理推荐）**：
   ```powershell
   # 下载并安装nvm-windows
   # 下载地址：https://github.com/coreybutler/nvm-windows/releases
   # 国内镜像：https://gitee.com/RubyMetric/nvm-cn/releases
   
   # 安装完成后
   nvm install lts
   nvm use lts
   ```
   
   **方式4：手动下载安装**：
   ```
   📥 请从以下地址下载Node.js安装包：
   
   官方下载：https://nodejs.org/zh-cn/download/
   国内镜像：https://npmmirror.com/mirrors/node/
   
   推荐版本：
   - LTS版本（长期支持）：v18.x 或 v20.x
   - 架构：根据您的系统选择x64或ARM64版本
   
   安装提示：
   1. 下载.msi安装包
   2. 运行安装程序，建议勾选"Add to PATH"
   3. 安装完成后重启终端
   ```

   **Mac系统**：
   
   **方式1：使用Homebrew（推荐）**：
   ```bash
   # 安装Node.js（LTS版本）
   brew install node@18
   
   # 或安装最新版本
   brew install node
   
   # 配置环境变量（如果安装了指定版本）
   echo 'export PATH="/usr/local/opt/node@18/bin:$PATH"' >> ~/.zshrc
   source ~/.zshrc
   ```
   
   **方式2：使用nvm（版本管理推荐）**：
   ```bash
   # 安装nvm
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
   
   # 国内镜像（如果官方源不可用）
   curl -o- https://gitee.com/mirrors/nvm/raw/v0.39.0/install.sh | bash
   
   # 安装完成后，重新加载配置
   source ~/.zshrc
   
   # 安装Node.js LTS版本
   nvm install --lts
   nvm use --lts
   nvm alias default lts/*
   ```
   
   **方式3：手动下载安装**：
   ```
   📥 请从以下地址下载Node.js安装包：
   
   官方下载：https://nodejs.org/zh-cn/download/
   国内镜像：https://npmmirror.com/mirrors/node/
   
   推荐版本：
   - LTS版本：v18.x 或 v20.x
   - 架构：根据您的Mac选择x64（Intel）或arm64（Apple Silicon）版本
   ```

3. **配置npm国内镜像源**（提升下载速度）：
   ```bash
   # 检查当前npm源
   npm config get registry
   
   # 配置淘宝镜像源
   npm config set registry https://registry.npmmirror.com
   
   # 验证配置
   npm config get registry
   
   # 测试npm是否正常工作
   npm --version
   ```

4. **验证安装结果**：
   ```bash
   # 检查Node.js版本
   node --version
   
   # 检查npm版本
   npm --version
   
   # 测试运行简单的Node.js代码
   node -e "console.log('Node.js is working!')"
   ```

#### 1.3 包管理器（npm/yarn/pnpm）

**检测步骤**：

1. **检查npm是否可用**（随Node.js安装）：
   ```bash
   npm --version
   ```
   - ✅ 输出版本号 → npm已安装
   - ❌ 命令不存在 → 异常情况（npm应该随Node.js一起安装）

2. **检查yarn是否可用**：
   ```bash
   yarn --version
   ```
   - ✅ 输出版本号 → yarn已安装
   - ❌ 命令不存在 → yarn未安装

3. **检查pnpm是否可用**：
   ```bash
   pnpm --version
   ```
   - ✅ 输出版本号 → pnpm已安装
   - ❌ 命令不存在 → pnpm未安装

4. **安装yarn（如果需要）**：
   
   **使用npm全局安装**：
   ```bash
   npm install -g yarn
   
   # 验证安装
   yarn --version
   ```
   
   **配置yarn国内镜像**：
   ```bash
   # 配置淘宝镜像
   yarn config set registry https://registry.npmmirror.com
   
   # 验证配置
   yarn config get registry
   ```

5. **安装pnpm（如果需要）**：
   
   **使用npm全局安装**：
   ```bash
   npm install -g pnpm
   
   # 验证安装
   pnpm --version
   ```
   
   **或使用独立安装脚本**：
   
   **Windows（PowerShell）**：
   ```powershell
   iwr https://get.pnpm.io/install.ps1 -useb | iex
   ```
   
   **Mac/Linux**：
   ```bash
   curl -fsSL https://get.pnpm.io/install.sh | sh -
   ```
   
   **配置pnpm国内镜像**：
   ```bash
   # 配置淘宝镜像
   pnpm config set registry https://registry.npmmirror.com
   
   # 验证配置
   pnpm config get registry
   ```

6. **升级npm到最新版本**（可选）：
   ```bash
   # 升级npm
   npm install -g npm@latest
   
   # 验证版本
   npm --version
   ```

**检测步骤**：

   # 创建pip配置目录

   index-url = https://pypi.tuna.tsinghua.edu.cn/simple

   # 创建pip配置目录

   pip config list
   ```

   ```

### 阶段2：浏览器和扩展检测

#### 2.1 Chrome/Edge浏览器检测

**检测步骤**：

1. **检查Chrome是否安装**：

   **Windows系统**：
   ```powershell
   # 检查常见安装路径
   $chromePaths = @(
       "$env:ProgramFiles\Google\Chrome\Application\chrome.exe",
       "${env:ProgramFiles(x86)}\Google\Chrome\Application\chrome.exe",
       "$env:LOCALAPPDATA\Google\Chrome\Application\chrome.exe"
   )
   
   $chromeInstalled = $false
   foreach ($path in $chromePaths) {
       if (Test-Path $path) {
           $chromeInstalled = $true
           Write-Host "Chrome已安装: $path"
           break
       }
   }
   ```

   **Mac系统**：
   ```bash
   # 检查Chrome应用程序
   if [ -d "/Applications/Google Chrome.app" ]; then
       echo "Chrome已安装"
   else
       echo "Chrome未安装"
   fi
   ```

2. **检查Edge是否安装**（Windows 10/11自带）：

   **Windows系统**：
   ```powershell
   # 检查Edge路径
   $edgePaths = @(
       "$env:ProgramFiles\Microsoft\Edge\Application\msedge.exe",
       "${env:ProgramFiles(x86)}\Microsoft\Edge\Application\msedge.exe"
   )
   
   $edgeInstalled = $false
   foreach ($path in $edgePaths) {
       if (Test-Path $path) {
           $edgeInstalled = $true
           Write-Host "Edge已安装: $path"
           break
       }
   }
   ```

   **Mac系统**：
   ```bash
   # 检查Edge应用程序
   if [ -d "/Applications/Microsoft Edge.app" ]; then
       echo "Edge已安装"
   else
       echo "Edge未安装"
   fi
   ```

3. **如果Chrome和Edge都未安装，提供安装指导**：

   **安装Chrome**：
   ```
   📥 Google Chrome下载地址：
   
   官方下载：https://www.google.com/chrome/
   
   安装提示：
   1. 下载适合您系统的安装包
   2. 运行安装程序
   3. 安装完成后即可使用
   ```

   **使用Edge**（Windows推荐）：
   ```
   💡 Windows 10/11自带Microsoft Edge浏览器
   
   如果Edge未安装或需要更新：
   官方下载：https://www.microsoft.com/zh-cn/edge
   ```

4. **检查浏览器版本**（确保版本足够新）：
   
   **Chrome版本检测**：
   - Windows: 在Chrome地址栏输入 `chrome://version`
   - Mac: Chrome菜单 > 关于Google Chrome
   
   **Edge版本检测**：
   - Windows: 在Edge地址栏输入 `edge://version`
   - Mac: Edge菜单 > 关于Microsoft Edge

#### 2.2 Cursor扩展检测

**检测步骤**：

1. **检查Live Server扩展是否安装**：

   **Windows系统**：
   ```powershell
   # 检查Cursor扩展目录
   $cursorExtPath = "$env:USERPROFILE\.cursor\extensions"
   $liveServerPattern = "ritwickdey.liveserver-*"
   
   if (Test-Path $cursorExtPath) {
       $liveServer = Get-ChildItem -Path $cursorExtPath -Filter $liveServerPattern -Directory
       if ($liveServer) {
           Write-Host "Live Server已安装: $($liveServer.Name)"
       } else {
           Write-Host "Live Server未安装"
       }
   }
   ```

1. **检查默认Shell**：
   ```bash
   # 查看当前Shell
   echo $SHELL
   
   # macOS Catalina及以后版本默认使用zsh
   # 如果使用bash，建议切换到zsh
   chsh -s /bin/zsh
   ```

2. **配置zsh配置文件**：
   ```bash
   # 编辑~/.zshrc文件
   cat >> ~/.zshrc <<EOF
   
   # Node.js configuration
   export PATH="/usr/local/bin:$PATH"
   
   # Homebrew configuration (Apple Silicon)
   if [[ `uname -m` == 'arm64' ]]; then
       eval "$(/opt/homebrew/bin/brew shellenv)"
   fi
   
   # Git aliases
   alias gs='git status'
   alias ga='git add'
   alias gc='git commit'
   alias gp='git push'
   
   # Python aliases
   alias python='python3'
   alias pip='pip3'
   
   EOF
   
   # 重新加载配置
   source ~/.zshrc
   ```

3. **配置命令行工具路径**（Apple Silicon Mac）：
   ```bash
   # Homebrew安装路径不同
   # Intel Mac: /usr/local
   # Apple Silicon Mac: /opt/homebrew
   
   # 检查架构
   arch_name="$(uname -m)"
   if [ "${arch_name}" = "arm64" ]; then
       echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc
   fi
   ```

## 注意事项

1. **管理员权限**：
   - Windows系统安装某些软件可能需要管理员权限
   - 建议以管理员身份运行PowerShell或命令提示符
   - Mac系统某些操作可能需要输入管理员密码

2. **网络要求**：
   - 下载和安装软件需要网络连接
   - 如果国际网络不稳定，建议使用 `--china-mirror` 选项
   - 某些镜像源可能需要时间同步，如遇问题可尝试其他镜像

3. **系统兼容性**：
   - Windows: 支持Windows 10（1809及以上）、Windows 11
   - Mac: 支持macOS 10.15 Catalina及以上版本
   - 某些旧系统可能无法安装最新版本的软件

4. **Apple Silicon Mac特殊说明**：
   - Apple Silicon（M1/M2/M3）Mac使用ARM64架构
   - Homebrew安装路径为 `/opt/homebrew`（不同于Intel Mac的 `/usr/local`）
   - 某些npm包可能需要Rosetta 2转译，首次安装会提示

5. **终端重启**：
   - 安装软件后，某些命令可能需要重启终端才能生效
   - 如果命令不可用，请关闭并重新打开终端
   - 或在当前终端重新加载配置文件（Mac: `source ~/.zshrc`）

6. **安全提示**：
   - 从官方渠道下载软件，避免使用第三方下载站
   - 配置镜像源时，确保镜像站是可信的
   - Git凭据请妥善保管，不要在公共场所泄露

7. **故障排查**：
   - 如果某个工具检测失败，可以单独重新安装
   - 使用 `--verbose` 选项查看详细日志
   - 检查PATH环境变量是否正确配置
   - 某些杀毒软件可能会误报，需要添加信任
